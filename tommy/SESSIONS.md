# Tommy Sessions Guide

Tommy (@TommyPickles999) runs 3 daily sessions on X, each with a different purpose. All sessions are orchestrated by OpenClaw cron jobs running in isolated sessions.

---

## 🔍 Explore Session (11:00 AM ET)

**Purpose:** Discover content matching Victor's interests, build engagement organically.

**Cron Job:** `Tommy Explore Session` (id: `6bcaf82e-...`)
**Schedule:** Daily at 11:00 AM ET

### How It Works

1. **Load Config** — Reads runtime config from MongoDB `tommy_config`
2. **Collect Posts** — `collect.js --type explore` fetches timeline + interest-based search posts
3. **Score Posts** — Agent scores each post using Tommy's scoring skill (interest match, quality, engagement)
4. **Engagement** — `engage.js` likes high-scoring posts, follows quality authors, optionally quote-RTs the best find
5. **Report** — Summarizes activity

### Key Scripts

- `src/collect.js --type explore` — Collect timeline + interest search posts
- `src/engage.js <scores> <collected>` — Execute likes, follows, quote-RTs
- `src/interest-matcher.js` — Score posts against Victor's interests

### Config (MongoDB `tommy_config`)

```json
{
  "explore": {
    "maxLikes": 10,
    "maxFollows": 3,
    "maxSearches": 5,
    "minScoreToPost": 8
  }
}
```

### Cost

~$0.27/session (API search costs)

## 📈 Market Session (12:30 PM ET, weekdays)

**Purpose:** Engage with stock market / finance content, post the daily DSP pick with social image.

**Cron Job:** `Tommy Market Session` (id: `118afae2-...`)
**Schedule:** Weekdays at 12:30 PM ET

### How It Works

1. **Load Config** — Reads runtime config from MongoDB `tommy_config`
2. **Collect Market Posts** — `collect-market.js` searches for stock tickers Tommy follows, fetches DSP pick + social image
3. **Score Posts** — Agent scores with MARKET-SPECIFIC criteria (earnings reactions, analyst upgrades, novel thesis = 9-10; generic commentary = 4-6)
4. **Generate Quote-RTs** — For posts scoring >= `market.minScoreToQuoteRT`, Tommy writes punchy quote-RTs
5. **Execute Engagement** — `engage.js` with `--quote-rts-file` for batch quote-RTs
6. **Post DSP Pick** — `post-pick.js --text "..." --image-path "/tmp/dsp-social-YYYY-MM-DD.png"` posts the daily pick with social image
7. **Self-Reply** — After pick post, Tommy self-replies with thesis link: `dailystockpick.ai/pick/YYYY-MM-DD`
8. **Report** — Summarizes tickers, scores, engagements, DSP post

### Key Scripts

- `src/collect-market.js` — Collect stock market posts + DSP pick + social image
- `src/engage.js <scores> <collected> --quote-rts-file <file>` — Execute engagement + batch quote-RTs
- `src/post-pick.js --text "..." --image-path "..."` — Post DSP pick with image + self-reply

### Config (MongoDB `tommy_config`)

```json
{
  "market": {
    "minScoreToQuoteRT": 8,
    "maxQuoteRTs": 3,
    "maxSearches": 8,
    "maxSearchResults": 15
  },
  "posting": {
    "enabled": true,
    "maxPickPostsPerDay": 1,
    "maxQuoteRTsPerDay": 2,
    "maxWords": 50
  }
}
```

### Cost

~$0.27/session (API search costs) + social image is pre-generated by DSP

## 🔥 Hot Take Session (7:30 PM ET)

**Purpose:** Generate genuinely original, thought-provoking content to grow followers through intellectual engagement.

**Cron Job:** `Tommy Hot Take Session` (id: TBD)
**Schedule:** Daily at 7:30 PM ET

### How It Works

1. **Load Config** — Reads runtime config from MongoDB `tommy_config`
2. **Determine Inspiration Source** — Rotates day-by-day through 3 sources:
   - **Day 1: Web Search** — Tommy picks an interesting topic, explores it via web search, forms an original angle
   - **Day 2: X Trending** — Fetches personalized trending topics via API, picks the most intellectually interesting one
   - **Day 3: Victor's Archive** — Searches @victor\_\_vector's post history for interesting old ideas to expand on
3. **Collect Inspiration** — `collect-hottake.js --mode <source>` gathers raw material
4. **Generate Hot Take** — Agent develops a genuinely ORIGINAL take. Not a summary — a new idea built on the seed.
5. **Plan Engagements** — Agent creates a plan with up to `maxEngagements` interactions
6. **Execute Engagements** — `execute-hottake.js` carries out the plan:
   - 🎨 **Image Post** — Generate AI image (OpenAI) + post with hot take caption
   - 🔄 **Quote-RT** — Find a popular related post, quote-RT with unique angle
   - 💬 **Thread Reply** — Find an active conversation, contribute the hot take
7. **Report** — Summarizes inspiration source, the take, engagements, links

### Key Scripts

- `src/collect-hottake.js --mode <web_search|x_trending|victor_archive>` — Collect inspiration
- `src/execute-hottake.js --plan /tmp/tommy-hottake-plan.json` — Execute engagements
- `src/gen-hottake-image.js` — Generate images via OpenAI API

### Config (MongoDB `tommy_config`)

```json
{
  "hotTake": {
    "enabled": true,
    "maxEngagements": 3,
    "inspirationRotation": ["web_search", "x_trending", "victor_archive"],
    "victorUsername": "victor__vector",
    "minFollowersForQuoteRT": 5000,
    "imageGenEnabled": true,
    "imageModel": "gpt-image-1",
    "imageSize": "1024x1024"
  }
}
```

### Prompt Philosophy

The Hot Take prompt encourages Tommy to be:

- **Weird** — Not corporate, not safe. Psychedelic, futuristic, contrarian.
- **Deep** — Genuine insight, not controversy for clicks.
- **Original** — "Don't summarize what you read. Find the thing no one else noticed."
- **Timely** — The take should feel like it belongs to THIS moment.
- **Shareable** — The kind of post that makes someone stop scrolling and think.

### Cost

~$0.04-0.08/session (OpenAI image generation) + negligible API costs

## Shared Infrastructure

### Runtime Config

All sessions read from MongoDB `tommy_config` collection. Dashboard Config page can modify values at runtime. Scripts use `config.loadRuntimeConfig(db)` to merge database values over defaults.

### Daily Caps (Shared Across Sessions)

- `posting.maxQuoteRTsPerDay: 2` — Total quote-RTs across ALL sessions
- `posting.maxPickPostsPerDay: 1` — DSP pick posts per day

### X API Client

All sessions use `src/x-client.js` which provides:

- OAuth 1.0a authentication
- Rate limiting with delays
- Exponential backoff on 429s
- API usage logging to MongoDB

### Tommy's Voice

Tommy is authentic, punchy, max 15 words for quote-RTs. No hashtags. Adds genuine commentary, never just restates. See `~/clawd/skills/tommy-scoring/SKILL.md` for full voice guidelines.
